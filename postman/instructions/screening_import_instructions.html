Screening API Postman Collection Import Instructions

Step 1: Download Postman

If you don't have Postman installed, download and install it from https://www.postman.com/downloads/

Step 2: Import the Collection

  1. Open Postman
  2. Click the "Import" button in the top left corner
  3. Select the "Files" tab
  4. Click "Upload Files" and select the Screening_API_Collection.json file
  5. Click "Import"

Step 3: Import the Environment

  1. In Postman, click the "Import" button again
  2. Select the "Files" tab
  3. Click "Upload Files" and select the Screening_API_Environment.json file
  4. Click "Import"

Step 4: Select the Environment

  1. In the top right corner of Postman, click the environment dropdown
  2. Select "Screening API Environment" from the list

Step 5: Set Up Date Variables

You need to set up the date variables. You can do this in two ways:

Option 1: Manual Setup

  1. Click the "Environment quick look" button (eye icon) in the top right
  2. Set the date variable to today's date in YYYY-MM-DD format (e.g., "2023-12-01")
  3. Set the displayDate variable to a human-readable date (e.g., "December 1, 2023")
  4. Click outside the environment panel to close it

Option 2: Using the Script

  1. Click on the collection name "Screening API Collection" in the sidebar
  2. Click on the "Pre-request Scripts" tab
  3. Copy and paste the following script:
// Get today's date in YYYY-MM-DD format
const today = new Date();
const todayFormatted = today.toISOString().split('T')[0];

// Format display date (e.g., "December 25, 2023")
const options = { year: 'numeric', month: 'long', day: 'numeric' };
const displayDate = today.toLocaleDateString('en-US', options);

// Get day of week
const daysOfWeek = ['Sunday', 'Monday', 'Tuesday', 'Wednesday', 'Thursday', 'Friday', 'Saturday'];
const dayOfWeek = daysOfWeek[today.getDay()];

// Set environment variables
pm.environment.set('date', todayFormatted);
pm.environment.set('displayDate', displayDate);
pm.environment.set('dayOfWeek', dayOfWeek);

console.log('Date variables set:');
console.log('date:', todayFormatted);
console.log('displayDate:', displayDate);
console.log('dayOfWeek:', dayOfWeek);
  • Click "Save"
  • Run any request in the collection to trigger the script

    • Step 6: Find a Movie

    1. Expand the "Movies" folder in the collection
    2. Click on "Get Now Playing Movies"
    3. Click "Send" to execute the request
    4. From the response, copy the ID of a movie you want to use for testing
    5. Click the "Environment quick look" button (eye icon) in the top right
    6. Set the movieId variable to the copied ID

    Step 7: Find a Cinema

    1. Expand the "Cinemas" folder
    2. Click on "Get Cinemas by Movie and Date"
    3. Click "Send" to execute the request
    4. From the response, copy a cinema ID
    5. Set the cinemaId variable to the copied ID

    Step 8: Get Screenings

    1. Expand the "Screenings" folder
    2. Click on "Get Screenings by Movie, Cinema and Date"
    3. Click "Send" to execute the request
    4. The response will show all screenings for the selected movie at the selected cinema on the specified date
    5. From the response, copy a screening ID
    6. Set the screeningId variable to the copied ID

    Step 9: Get Screening Details

    1. Click on "Get Screening by ID"
    2. Click "Send" to execute the request
    3. The response will show detailed information about the selected screening
    4. Click on "Get Available Seats for Screening" to see available seats

    Step 10: Admin Operations (Optional)

    1. Expand the "Authentication" folder
    2. Click on "Login Admin"
    3. Modify the credentials if needed
    4. Click "Send" to execute the request
    5. From the response, copy the token value
    6. Set the adminToken variable to the copied token
    7. Now you can use the admin endpoints to create, update, or delete screenings

    Note:

    If you're not getting any results, try the following:

    • Use a different movie ID (some movies might not have any screenings)
    • Try a different date (the movie might not be showing on the selected date)
    • Check that the date format is correct (YYYY-MM-DD)
    • Verify that the API base URL is correct
