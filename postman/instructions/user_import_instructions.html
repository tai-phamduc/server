User API Postman Collection Import Instructions

Step 1: Download Postman

If you don't have Postman installed, download and install it from https://www.postman.com/downloads/

Step 2: Import the Collection

  1. Open Postman
  2. Click the "Import" button in the top left corner
  3. Select the "Files" tab
  4. Click "Upload Files" and select the User_API_Collection.json file
  5. Click "Import"

Step 3: Import the Environment

  1. In Postman, click the "Import" button again
  2. Select the "Files" tab
  3. Click "Upload Files" and select the User_API_Environment.json file
  4. Click "Import"

Step 4: Select the Environment

  1. In the top right corner of Postman, click the environment dropdown
  2. Select "User API Environment" from the list

Step 5: Register or Login

  1. In the collection, expand the "Authentication" folder
  2. To create a new user, click on the "Register User" request
  3. Modify the email and password in the request body if needed
  4. Click "Send" to execute the request
  5. From the response, copy the token value
  6. Click the "Environment" quick look button (eye icon) in the top right
  7. Set the "userToken" variable to the copied token
  8. Also set the "userId" variable to the _id value from the response

Alternatively, if you already have user credentials:

  1. Click on the "Login User" request
  2. Enter your email and password in the request body
  3. Click "Send" to execute the request
  4. Copy the token and set the "userToken" variable

Step 6: Test User Profile Endpoints

  1. Expand the "User Profile" folder
  2. Click on "Get User Profile" and click "Send" to see your profile information
  3. Try "Update User Profile" to modify your profile details

Step 7: Test Booking Functionality

To test booking functionality, you'll need valid movie and showtime IDs:

  1. Use the Movie API Collection to get valid movie and showtime IDs
  2. Set the "movieId" and "showtimeId" environment variables
  3. Expand the "User Bookings" folder
  4. Use "Create Booking" to make a new booking
  5. From the response, copy the booking ID and set the "bookingId" variable
  6. Use "Get User Bookings" to see all your bookings

Note:

Some endpoints require authentication with a valid JWT token. Make sure you've completed Step 5 before testing these endpoints.

Admin endpoints require an admin token. You'll need admin credentials to access these endpoints.
